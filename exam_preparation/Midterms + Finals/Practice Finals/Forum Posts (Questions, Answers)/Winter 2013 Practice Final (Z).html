Re: Winter 2013 Practice Final (Z)
by Abram Hindle - Tuesday, 15 December 2015, 1:29 AM
 

> In design patterns question 2, "operations" was circled. Does this mean that the command pattern would be better? What about the part that mentions composing operations together, does that hold any merit?

Yes. Command is a good hint if operations are mentioned. Operations are verb nounified -- made actionable objects.


> For MVC and Observer pattern, the model knows the data for all the views. But should we mention that because it says do not define model?

The model has the data for the views, the model doesn't know what the views want. The views know what they want. The model only knows it has observers and it can call update on them.

> Also for the UML class diagram, I'm assuming there should be aggregation in addition to the association?

You can have more than 1 link between 2 objects if it is important. Yes.

> In the template method part, can the subclasses that inherit from the class have the same method?

The should override the method.

> For the testing part, when you circled httpPost, does that meant there should be a void httpPost method?

It means you were supposed to implement that httpPost method.

> For the refactoring part, do we need to list the methods of the subclass?

If it's relevant to the pattern. Template has abstract operations, those are important.
